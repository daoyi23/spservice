# 背景
目前市场有很多485串口服务器软件，但是基本都和硬件进行了强绑定，最近看到一个开源的串口服务器软件，支持MQTT、HTTP、Modbus、TCP/UDP等协议，并且只需要改配置文件，无需改代码就可以适配到其它的硬件平台。

## 2	端口管理
### 2.1	端口配置
#### 2.1.1	编辑串口
##### 2.1.1.1	485口编辑
点击【编辑】按钮，可对不同串口通道进行编辑，编辑内容：485口相关参数波特率，数据位想，校验位等，同时设置通道是否启用，以及通道是否采用轮询方式采集数据。如果设置自动采集数据，需要先添加轮询命令。
![在这里插入图片描述](https://i-blog.csdnimg.cn/direct/8793adcf3b59478fba63f26a71db4afe.png)
##### 2.1.1.2	添加网口
点击【添加网口】按钮，根据实际情况选择需要的类似，添加完成后，重启系统生效。
可添加网口模式：

> 1)	TCPClent：TCP客户端模式 
> 2)	TCPServer：TCP服务模式
>  3)	UDPServer：UDP服务端模式
> 4)	ComServer：串口模式，这个主要用在设备有多个串口设备的，将一个串口数据，转发到另外一个串口上面。
> 5)	MQTTClient：MQTT协议通讯数据
>  6)	HTTPClient：HTTP协议通讯数据，只支持轮询查询，自动上报到服务器
> 7)	ModbusTcpClient：ModbusTCP协议，客户端模式
> 8)	ModbusTcpServer：ModbusTCP协议，服务端模式

**备注：**
   对轮询查询，自动上报，需要根据传输类型，进行一点小的私有定制。
1)	TCP，UDP等二进制流方式，采用私有协议定制，在透传数据前增加消息头，消息头定义如下
> MessageHead{
>  UInt16 m_Code;     //在添加自动采集命令时，用户自己定义的编码，编码必须唯一
>  UInt16 m_DLen;     //数据长度，采集到的数据长度 }

2)	HTTP，MQTT上报，**包体采用JSON格式上报**，code为用户在添加自动采集时，用户自定义的编码，编码唯一，
> 	   	   data为透出的数据16进制串
> 	   	   { 			
> 	   	 "code": 25,      			
> 	   	  "data": "01040A0D"
> 	   	   }

# 编译方法

## Windows编译

1. 进入代码目录下的sdk目录，双击执行update_sdk.bat，下载windows版本sdk包。
1. 进入到代码web/code，执行cnpm install，安装web界面编译需要的库。(系统需要安装nodejs)
2. 下载完成后，双击执行install_win.bat，安装sdk库
3. 进入代码下的src\spservice目录，用VS2019打开工程进行编译

## Linux编译

1. 先需要在代码目录sdk目录执行对应的硬件平台脚本，**例如：sh update_xxx_xxx_xxxx.sh**，下载对应的sdk库。

1. 进入到代码web/code，执行cnpm install，安装web界面编译需要的库。(系统需要安装nodejs)

2. 在进入到代码目录的build目录执行对应的硬件平台脚本，**例如：. build_xxx_xxx_xxxx.sh** ，脚本支持参数：

   -re :全量重新编译

   -debug： 编译debug版本

   -release：编译release版本，默认编译release

3. 编译完成后，版本会自动打包到version目录
